# 🚀 Быстрый старт

## Запуск бота за 3 шага

### Шаг 1: Установка зависимостей

```bash
# Активируйте виртуальное окружение
.venv1\Scripts\activate

# Установите зависимости (если еще не установлены)
pip install -r requirements.txt
```

### Шаг 2: Настройка

1. Убедитесь, что файл `.env` содержит ваш BOT_TOKEN:
```
BOT_TOKEN=your_bot_token_here
```

2. Измените ADMIN_ID в `main.py` на ваш Telegram ID:
```python
ADMIN_ID = 1121167993  # Замените на ваш ID
```

### Шаг 3: Запуск

```bash
python main.py
```

Бот запущен! ✅

## 🌐 Запуск Mini App (для тестирования)

### Вариант 1: Локальное тестирование с ngrok

1. **Запустите бота** (он автоматически запустит веб-сервер на порту 8080)

2. **В новом терминале запустите ngrok:**
```bash
ngrok http 8080
```

3. **Скопируйте HTTPS URL** из ngrok (например: `https://1234-5678.ngrok-free.app`)

4. **Обновите URL в `keyboards.py`:**
```python
WEBAPP_URL = "https://1234-5678.ngrok-free.app/webapp/index.html"
```

5. **Перезапустите бота**

6. **Откройте бота в Telegram** и нажмите "🎁 Открыть кейсы"

### Вариант 2: Деплой на Render/Heroku

Следуйте инструкциям в `SETUP_MINIAPP.md`

## 🐛 Решение проблем

### Бот не запускается
```bash
# Проверьте, что виртуальное окружение активировано
.venv1\Scripts\activate

# Переустановите зависимости
pip install -r requirements.txt --force-reinstall
```

### Mini App не открывается
1. Убедитесь, что WEBAPP_URL начинается с `https://`
2. Проверьте, что ngrok запущен
3. Перезапустите бота после изменения URL

### Имя показывает "Guest"
1. Напишите боту /start перед открытием Mini App
2. Проверьте, что веб-сервер запущен (порт 8080)
3. Откройте DevTools (F12) и проверьте Console на ошибки

### Баланс сбрасывается
1. Убедитесь, что файл `bot_users.db` существует
2. Проверьте логи в консоли
3. Откройте DevTools и проверьте Network tab
