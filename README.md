# 🎮 Case Clicker Bot - Hamster Kombat Style v3.1

Современный Telegram бот с Mini App в стиле Hamster Kombat. Кликайте, открывайте кейсы, улучшайте способности и зарабатывайте монеты!

## ✨ Возможности

### 🎮 Игровая механика (как в Hamster Kombat):
- 👆 **Тап-система** - кликайте и зарабатывайте монеты
- ⚡ **Энергия** - 1000 энергии, восстанавливается 1/сек
- 🎁 **Кейсы** - 4 типа кейсов с разными предметами
- ⬆️ **Улучшения** - увеличивайте силу клика, энергию и т.д.
- 🏆 **Достижения** - 6 достижений за различные действия
- 📊 **Уровни** - прокачивайте уровень и получайте опыт
- 💼 **Инвентарь** - собирайте предметы из кейсов
- 🎉 **Ежедневная награда** - получайте бонус каждый день

### 🤖 Функции бота:
- Автоматическая регистрация пользователей
- 📢 Рассылка сообщений всем пользователям
- 📊 Детальная статистика и аналитика
- 👥 Просмотр списка пользователей с пагинацией
- 👨‍💼 Система управления администраторами
- 🎨 Красивый интерфейс с кнопками
- 💾 Надежное сохранение данных в БД

## 🚀 Быстрый старт

### 1. Установка зависимостей:
```bash
# Активируйте виртуальное окружение
.venv1\Scripts\activate

# Установите зависимости
pip install -r requirements.txt
```

### 2. Настройка:

1. Создайте файл `.env` с вашим токеном:
```
BOT_TOKEN=ваш_токен_от_BotFather
```

2. В файле `main.py` замените `ADMIN_ID` на ваш Telegram ID (строка 66):
```python
ADMIN_ID = 1121167993  # Замените на ваш ID
```

3. Для Mini App обновите URL в `keyboards.py` (строка 5):
```python
WEBAPP_URL = "https://your-domain.com/webapp/index.html"
```

### 3. Запуск:

```bash
python main.py
```

Бот запущен! Веб-сервер автоматически запустится на порту 8080.

## Настройка Mini App

### Для локального тестирования:
1. Используйте ngrok или аналогичный сервис для создания публичного URL:
```bash
ngrok http 8080
```

2. Замените URL в `keyboards.py` на полученный от ngrok

### Для продакшена:
1. Разместите файлы из папки `webapp/` на вашем веб-сервере
2. Убедитесь, что сервер поддерживает HTTPS
3. Обновите URL в `keyboards.py`

## 📊 API Endpoints

Mini App использует следующие API endpoints:

- `POST /api/balance` - Получение баланса пользователя
- `POST /api/open_case` - Открытие кейса
- `POST /api/inventory` - Получение инвентаря
- `POST /api/stats` - Статистика открытий
- `POST /api/game_data` - Получение всех игровых данных
- `POST /api/save_game_data` - Сохранение игровых данных
- `POST /api/user_info` - Получение информации о пользователе

## 💾 База данных

### Таблицы SQLite (bot_users.db):
- `users` - Пользователи бота (user_id, username, first_name, last_name)
- `admins` - Администраторы
- `statistics` - Статистика событий
- `user_balance` - Баланс пользователей
- `user_game_data` - Игровые данные (уровень, опыт, энергия, клики)
- `user_upgrades` - Купленные улучшения
- `user_achievements` - Достижения пользователей
- `user_inventory` - Инвентарь предметов
- `case_openings` - История открытий кейсов

### Сохранение данных:
- ✅ Автосохранение каждые 10 секунд
- ✅ Сохранение при каждом действии
- ✅ Резервная копия в localStorage
- ✅ Данные не сбрасываются при перезапуске

## 🐛 Решение проблем

### Имя показывает "Guest"
1. Напишите боту /start перед открытием Mini App
2. Проверьте, что веб-сервер запущен (порт 8080)

### Баланс сбрасывается
1. Убедитесь, что файл `bot_users.db` существует
2. Проверьте логи в консоли
3. Откройте DevTools (F12) и проверьте Network tab

### Mini App не открывается
1. Проверьте, что WEBAPP_URL использует HTTPS
2. Убедитесь, что ngrok запущен
3. Перезапустите бота после изменения URL
